# Этап 2 — Задачи (ручной режим, без LLM)

> Сначала `plan/CONVENTIONS.md`, затем разделы ТЗ ниже.

## Цель

Реализовать поток постановки задач для `input_mode = private` **без LLM**:
участник вводит цели текстом, бот структурирует их простыми правилами (одна
строка / маркер списка = одна задача), показывает экран подтверждения с правкой.
Это проверяет поток «ввод → подтверждение → фиксация» до подключения LLM (§15, этап 2).

## Читать в ТЗ

- §5 (`week`, `task`), §6.2 (постановка задач — ручная ветка), §6.6 (`/tasks`),
  §2 (подтверждение участником, чувство собственности).

## Предусловия

- Этап 1 готов: онбординг, `member`, `dialog_state`, FSM.

## Задачи

1. **Модели** `week` и `task` из §5 (+ enum `source`, `status`, поля
   `parent_task_id`, `position`, `confirmed`) + миграция. Репозиторий-функции:
   `get_or_create_current_week`, `create_tasks`, `list_tasks_for_member_week`,
   `replace_tasks`, `set_confirmed`.
2. **Сервис** `app/services/extraction.py` (на этом этапе — без LLM):
   - функция `parse_manual_input(text) -> list[str]`: разбивает ввод на задачи
     по строкам/буллетам, чистит пустые, тримит. Это место позже заменится/дополнится
     LLM на этапе 4 (оставь чёткую точку расширения).
   - функция запуска приватного ввода: бот пишет «Какие у тебя цели на эту неделю?»,
     ставит `dialog_state.state = confirming_tasks`.
3. **Роутер `tasks`** (`app/bot/routers/tasks.py`):
   - приём текстового ответа в состоянии сбора целей → `parse_manual_input` →
     создать `task` (`source = manual`, `confirmed = false`, проставить `position`).
   - **экран подтверждения**: показать нумерованный список задач + инлайн-кнопки
     «✅ Всё верно» / «✏️ Поправить».
   - «Поправить» → бот просит прислать исправленный список текстом → пересобрать
     задачи (`replace_tasks`) → снова показать подтверждение.
   - «Всё верно» → `confirmed = true`, `state = idle`, тёплое подтверждение.
   - `/tasks`: показать текущий список задач недели со статусами.
4. Подключить роутер в `main.py`. Тон сообщений — напарник.

## Точки расширения (важно для этапа 4)

- `parse_manual_input` и «структурирование» должны быть отдельной функцией, чтобы
  на этапе 4 подменить её на LLM-структурирование без переписывания роутера.
- Экран подтверждения переиспользуется для `auto`-режима (этап 4) — сделай его
  переиспользуемым (принимает member+week, не зависит от способа ввода).

## Создаются/меняются файлы

`app/db/models.py` (+`week`,`task`), `app/db/repo.py`, миграция,
`app/services/extraction.py`, `app/bot/routers/tasks.py`,
`app/bot/keyboards.py` (кнопки подтверждения/правки), `app/main.py`.

## Definition of Done

- В режиме `private` участник вводит цели текстом → видит структурированный список.
- Экран подтверждения: «Всё верно» фиксирует (`confirmed=true`), «Поправить»
  позволяет переслать новый список и пересобрать.
- До подтверждения задачи помечены `confirmed=false` (не считаются зафиксированными).
- `/tasks` показывает текущий список.

## Локальная проверка

1. Участник с `input_mode=private`.
2. Запустить сбор целей (временная команда `/setgoals` для теста допустима, если
   джоба-планировщик ещё не готова — она появится на этапе 3).
3. Ввести многострочный список → проверить разбиение и экран подтверждения.
4. «Поправить» → переслать новый список → подтверждение обновилось.
5. «Всё верно» → `confirmed=true` в БД. `/tasks` показывает список.

## Заметка для следующего этапа

Сбор целей пока дёргается вручную/командой. На этапе 3 появится планировщик —
именно он будет инициировать постановку задач и чек-ин по расписанию.
