# Этап 6 — Итог недели и витрина Google Sheets

> Сначала `plan/CONVENTIONS.md`, затем разделы ТЗ ниже.

## Цель

Реализовать формирование итога недели по маркеру `[REPORT_READY]`, принцип
«участник видит сводку первым», подтверждение отправки и маршрутизацию по
`member.visibility`, плюс запись финальных сводок в Google Sheets.

## Читать в ТЗ

- §2 (приватность, участник первым видит итог), §5 (`summary`), §6.5 (итог и
  маршрутизация), §9 (промпт 2 — маркер `[REPORT_READY]` и резюме), §11 (Sheets),
  §12 (`GOOGLE_SERVICE_ACCOUNT_JSON`).

## Предусловия

- Этап 5 готов: трекинг-диалог (промпт 2), декомпозиция, голос.

## Задачи

1. **Модель** `summary` из §5 (+enum `shared_scope`) + миграция. Repo:
   `create_summary`, `list_summaries_for_member`.
2. **Сервис** `app/services/summary.py`:
   - детектор маркера: когда трекинг-LLM (промпт 2) вернул `[REPORT_READY]` —
     **отделить** диалоговую часть от резюме (всё после маркера = резюме для ведущего).
   - сформировать `member_text` (что видит участник) и `facilitator_text`.
   - показать участнику **его сводку первой** + кнопка «Отправить по моим
     настройкам видимости?» (Да/Нет).
   - после «Да» — сохранить `summary` и маршрутизировать по `member.visibility`:
     - `group` → запись в Sheets-витрину (+опц. пост в групповой чат);
     - `facilitator` → личное сообщение ведущему (`group.facilitator_chat_id`);
     - `private` → никуда не уходит, хранится только у участника
       (`shared_scope=none`/`private`).
   - «Нет» → не отправлять, пометить как приватную.
3. **Sheets** (`app/services/sheets.py`) через `gspread` (service account, §11):
   - писать **только финальные сводки** с учётом видимости; `private`-участники
     **не** попадают в групповой лист.
   - Sheets — витрина на чтение; бот не читает из неё задачи (кроме опц. канала
     ручной вставки транскрипта из §10).
   - аккуратная обработка отсутствия `group.sheet_id` (просто не пишем).
4. Подключить вызов формирования итога в конце чек-ин-диалога (этап 5).

## Создаются/меняются файлы

`app/db/models.py` (+`summary`), `app/db/repo.py`, миграция,

## Definition of Done

- Финальную сводку участник видит **первым**; до подтверждения никуда не уходит.
- После «Да» сводка уходит строго по `visibility`; `private` — никуда (только хранение).
- В Sheets попадают только финальные сводки, приватные исключены.
- `summary` сохраняется с корректным `shared_scope`.

## Локальная проверка

1. Подготовить service account + тестовую таблицу, `sheet_id` у группы.
2. Пройти чек-ин до `[REPORT_READY]` → участник получает свою сводку первым.
3. Проверить три ветки видимости:
   - `group` → строка появилась в Sheets;
   - `facilitator` → пришло сообщение ведущему;
   - `private` → никуда не ушло, но `summary` есть в БД.
4. Убедиться, что приватная сводка не попала в групповой лист.
