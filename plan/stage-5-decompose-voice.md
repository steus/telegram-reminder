# Этап 5 — Декомпозиция по согласию + голос

> Сначала `plan/CONVENTIONS.md`, затем разделы ТЗ ниже.

## Цель

Добавить трекинг-диалог LLM (промпт 2), опт-ин декомпозицию задачи на 2–4 шага с
переносом в следующую неделю, и обработку голосовых сообщений через whisper.cpp.

## Читать в ТЗ

- §6.3 (голос в чек-ине), §6.4 (затык + согласие, декомпозиция), §9 (промпт 2),
  §5 (`task`: `source=decomposed`, `parent_task_id`, `week_id`=следующая;
  `dialog_state.state=decomposing`), §3/§14 (whisper.cpp), §12 (`WHISPER_*`).

## Предусловия

- Этап 4 готов: `ask_llm`+фолбэк, промпты, чек-ин на кнопках.

## Задачи

1. **Голос** (`app/services/voice.py`):
   - принять `message.voice`, ограничить максимальную длительность (§7),
     скачать файл → прогнать через `whisper.cpp` (путь к бинарю/модели из `.env`,
     `WHISPER_MODE=local`) → вернуть текст. Опция `WHISPER_MODE=api` (Whisper API).
   - блокирующий вызов whisper — через `run_in_executor`/subprocess async, не
     блокировать event loop.
   - В роутерах чек-ина/целей: голос обрабатывается «как текстовый ответ».
2. **LLM-трекинг в чек-ине** (промпт 2): если участник отвечает текстом/голосом
   (а не только тапает кнопки) — подключить `ask_llm` с промптом 2 и буфером
   диалога (`dialog_state.context_json`, последние N реплик). Распарсить статусы
   по задачам, обновить `task.status`.
3. **Декомпозиция** (`app/services/decompose.py`) — строго по согласию (§6.4):
   - при `status=stuck` бот спрашивает «Помочь разбить на шаги?» (кнопки Да/Нет).
   - «Да» → `state=decomposing`, LLM предлагает 2–4 конкретных шага (максимум один
     уточняющий вопрос). Шаги конкретны под ситуацию, не общие.
   - участник правит/подтверждает → шаги сохраняются как `task`
     (`source=decomposed`, `parent_task_id`=исходная, `week_id`=**следующая** неделя),
     исходная задача → `status=decomposed`.
   - «Нет» → никакого давления, остаёмся со `stuck`.
4. **Роутер `decompose`** (`app/bot/routers/decompose.py`) + подключение в `main.py`.
5. Тон: тёплый напарник, без менторства и допроса (см. промпт 2).

(whisper.cpp бинарь+модель как том или сборка; либо `WHISPER_MODE=api`),
`app/db/repo.py` (создание подзадач, перенос недели).

## Definition of Done

- Голосовой ответ расшифровывается и обрабатывается как текст.
- При `stuck` бот **предлагает** помощь; декомпозиция запускается только при «Да».
- Шаги конкретны, сохраняются в **следующую** неделю с `source=decomposed` и
  `parent_task_id`; исходная задача → `decomposed`.
- LLM-трекинг корректно проставляет статусы из свободного ответа.

## Локальная проверка

1. Настроить whisper (локальный бинарь+модель или API-режим).
2. Отправить голосовое в чек-ине → текст распознан, статусы проставлены.
3. Пометить задачу `stuck` → «Помочь разбить?» → «Нет» (ничего не дробится) и
   отдельно «Да» (получить 2–4 шага, подтвердить).
4. Проверить в БД: подзадачи `source=decomposed`, `week_id`=следующая неделя,
   исходная `status=decomposed`.

## Заметка по whisper в Docker

Реши и зафиксируй в `PROGRESS.md`: собирать whisper.cpp в образе (тяжелее образ,
автономно) либо монтировать готовый бинарь+модель томом (легче образ). Для слабого
VPS — допустимо сразу `WHISPER_MODE=api`.
